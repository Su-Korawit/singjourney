# ADR 2026-06-29 — การวางกรอบ "การมีส่วนร่วมชุมชน / LINE OA" ในเอกสารส่งประกวด

> รอบ grill-with-docs (แก้ Form 1-2 + infographic ตามข้อความครู สำหรับ I-New Gen Award 2027)
> สถานะ: Accepted

## บริบท / ปัญหา

ข้อความที่ครูเพิ่ม (ส่วน 5.1-5.5 และข้อ 6) ดัน **"การมีส่วนร่วมของชุมชนผ่าน LINE Official Account"** ให้เป็นแก่นนวัตกรรม (ชุมชนส่งข้อมูลพื้นที่จริง → AI ประมวลผล) และเขียนในเชิงว่าเกิดขึ้นแล้ว

แต่แอป MVP ปัจจุบันมี decision ตรงข้าม (commit `c399851`, `27d42c0`): เป็น **"no-LINE, no-human-in-the-loop MVP"** — ถอดปุ่ม LINE Login, ตัดคำกล่าวอ้าง "ตลาดอัปเดตสถานะโดยเจ้าของร้านผ่าน LINE OA" ออกเพราะเป็นข้อมูลเท็จ และคำนวณสถานะเปิด-ปิดอัตโนมัติจากเวลาทำการแทน ส่วนบันทึกทริปใช้ localStorage ไม่ใช่ LINE Login

ความจริงเชิงเทคนิค: **backend LINE มีอยู่จริงและครบ** (`src/lib/line/*`, `api/line/webhook`, migration `0002_line_features.sql`, types override/reporter) แต่ **ยังไม่ wire เข้า UI / ยังไม่เปิด channel จริง** → "ออกแบบและพัฒนาฐานระบบแล้ว" จริง แต่ "ชุมชนใช้งานครบแล้ววันนี้" ไม่จริง

เกิดความตึงระหว่างคำสั่งผู้ใช้ 2 ข้อ: "เขียนตามข้อความครู" vs "รวมแบบรักษาความจริง" + รอบแรกเป็น **เอกสารเป็นหลัก** (Form 2 ห้ามเกิน 1 หน้า A4) ภาพแอปจริงตามมาทีหลัง

## Decision

ใช้แนวทาง **"วิสัยทัศน์ + สถานะชัด" (Vision + Clear Status)**:
1. คงนารราตีฟชุมชน/LINE OA/storytelling/ระบบนิเวศ ของครูไว้เต็มในส่วนเชิงวิสัยทัศน์ (5.x, ข้อ 6, ที่มา)
2. กำกับสถานะทุกจุดที่อ้างฟีเจอร์ ด้วยป้าย 2 ระดับ:
   - **(ทำงานจริงแล้ว)** = ใช้ได้ในแอปวันนี้ (AI วางแผน, แผนที่+ลากวาง, คำนวณค่าใช้จ่าย, เช็กอินไอเทม 3D, Watt's Up!, สถานะเปิด-ปิดอัตโนมัติ, บันทึกทริปในเครื่อง, ฐานข้อมูล 12 จุด ฯลฯ)
   - **(พัฒนาฐานระบบแล้ว · นำร่องเฟสถัดไป)** = backend พร้อม แต่ยังไม่เปิดใช้จริง (ชุมชนแจ้งผ่าน LINE OA, LINE Login ผูกบัญชี, แจ้งเตือนผ่าน LINE)
3. แก้คำกล่าวอ้างเท็จเดิมในฟอร์ม ("saved plans ผ่านบัญชี LINE", "ข้อมูลจริง 2 ชั้น") ให้ตรงความจริง
4. เขียน implement plan ไว้เพื่อให้คำว่า "เฟสถัดไป" เป็นจริงได้ก่อนรอบกรรมการดูแอป

## ทางเลือกที่ไม่เลือก
- **เขียนตามครูแบบไม่ติดป้าย** — เสี่ยงเคลมฟีเจอร์ที่ยังไม่มี ผิด integrity และพังตอนเดโม
- **ติดป้ายสถานะทุกจุดแบบตาราง** — ซื่อสัตย์สุด แต่ตัดทอนนารราตีฟ ทำให้ Form 2 ยาวเกิน 1 หน้า
- **ใส่ LINE OA เป็น roadmap ล้วน (อนาคต)** — ดูถ่อมเกินจริง ทั้งที่ backend ทำไว้แล้ว เสียจุดขายเชิงระบบนิเวศ
