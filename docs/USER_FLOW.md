## การตัดสินใจที่ล็อกแล้ว (จาก grilling)

| # | ประเด็น | ตัดสิน |
| - | ------- | ------ |
| 1 | จุดเริ่มหลักอันเดียว | **"ให้ AI วางแผนทริป"** — เป็นปุ่มใหญ่เด่นปุ่มเดียวบนหน้าแรก |
| 2 | จุดไปต่อหลังได้แผน | กด "ใช้แผนนี้" → ส่งเข้า **Roadmap แผนที่** อัตโนมัติ |
| 3 | District Map | ทำหน้าที่เป็น **ภาพ identity** บอกว่า "นี่คือสิงห์บุรี" ไม่แข่งกับปุ่ม AI (ยังคลิกได้แต่เป็นทางรอง) |
| 4 | ลายเซ็นดีไซน์สิงห์บุรี | **ปลาแม่ลา** (SVG วาดเอง ไม่ติดลิขสิทธิ์) ใช้ในป้าย/ปุ่ม/ตัวคั่น/loading |
| 5 | ป้ายบอกทาง | เพิ่มแถบ **"เที่ยวสิงห์บุรีใน 3 ขั้นตอน"** บนหน้าแรก = ป้ายบอก station แบบหน่วยงานที่ออกแบบดี |

## จุดที่แก้ในรอบนี้ (โค้ดจริง)

- **หน้าแรก** ([src/app/page.tsx](../src/app/page.tsx)): ป้ายฟังก์ชัน "เว็บวางแผนเที่ยวสิงห์บุรีด้วย AI" +
  ปลาแม่ลา · ปุ่ม CTA เหลือ **อันเดียวเด่นชัด** (ลด "ดู Roadmap" เป็นลิงก์รอง) ·
  เพิ่มแถบ 3 ขั้นตอน · **แก้การ์ดฟีเจอร์ที่ซ่อนรายละเอียดใต้ hover → แสดงตลอด** (มือถือ/จอสัมผัสแตะติดแล้ว)
- **หน้าวางแผน** ([src/app/plan/page.tsx](../src/app/plan/page.tsx)): ดัน ProfileQuiz (ขั้นที่ 1) ขึ้นบนสุด +
  หัวข้อชัด "ตอบ 3 ข้อ ให้ AI จัดทริปสิงห์บุรีให้คุณ" · ย้ายแผนตัวอย่างลงเป็นทางเลือกรอง
- **คอมโพเนนต์ใหม่** [src/components/brand/MaeLaFish.tsx](../src/components/brand/MaeLaFish.tsx): ปลาแม่ลา + แถบปลาว่าย
