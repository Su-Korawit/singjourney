## ส่วนที่ 2 รายละเอียด

> หมายเหตุสถานะ (เพื่อความถูกต้องของข้อมูลในเอกสารส่งประกวด): ฟีเจอร์ที่ระบุ **(ทำงานจริงแล้ว)** ใช้งานได้ในเว็บแอปปัจจุบัน · ส่วนที่ระบุ **(พัฒนาฐานระบบแล้ว · นำร่องเฟสถัดไป)** คือกลไกที่พัฒนาโครงสร้าง/ฝั่งเซิร์ฟเวอร์ไว้แล้ว เตรียมเปิดใช้กับชุมชนจริงในระยะถัดไป

**1. ที่มาของแนวคิด**
จังหวัดสิงห์บุรีเป็น "เมืองรอง" ที่มีแหล่งท่องเที่ยวเชิงประวัติศาสตร์ วัฒนธรรม และสายทำบุญที่มีคุณค่า (เช่น วัดพระนอนจักรสีห์ฯ อนุสาวรีย์วีรชนค่ายบางระจัน) แต่ขาดเครื่องมือดิจิทัลที่ช่วยให้นักท่องเที่ยววางแผนเที่ยวได้ง่ายและเหมาะกับตนเอง และขาดช่องทางให้ชุมชนเจ้าของพื้นที่นำเสนออัตลักษณ์ของตนเอง แพลตฟอร์มท่องเที่ยวที่มีอยู่ส่วนใหญ่ทำหน้าที่เป็น "ไกด์บุ๊กออนไลน์" ให้ผู้ใช้ค้นหาและจัดทริปเอง ทีมจึงคิดค้นและพัฒนา Sing Journey ขึ้นเอง ไม่ได้มองเป็นเพียง "เว็บ AI วางแผนเที่ยว" แต่เป็นนวัตกรรมที่ใช้ AI ลดช่องว่างระหว่างเทคโนโลยีกับชุมชน เชื่อมโยงนักท่องเที่ยว ชุมชน ผู้ประกอบการท้องถิ่น และ AI ให้ทำงานร่วมกันเป็นระบบนิเวศเดียว โดย AI เข้าใจผู้ใช้และวางแผนทริปให้อัตโนมัติ เพิ่มความสนุกด้วยเกมิฟิเคชัน และยกระดับความแม่นยำของข้อมูลด้วยการผสานข้อมูลเปิด-ปิดอัตโนมัติกับข้อมูลที่ชุมชนแจ้งจากพื้นที่จริง

**2. วัตถุประสงค์**
1. สร้างแพลตฟอร์มที่ใช้ AI วิเคราะห์ไลฟ์สไตล์ผู้ใช้และสร้างเส้นทางท่องเที่ยวจังหวัดสิงห์บุรีแบบเฉพาะบุคคล
2. รวมการนำเสนอ วางแผน และนำทางบนแผนที่จริงไว้ในที่เดียว แก้ไขได้ง่ายด้วยการลากวาง
3. ส่งเสริมการท่องเที่ยวเมืองรองและกระจายรายได้สู่ชุมชน วัด และร้านค้าท้องถิ่นผ่านระบบเกมิฟิเคชัน
4. เปิดให้ชุมชน วัด ตลาด และผู้ประกอบการท้องถิ่นมีส่วนร่วมอัปเดตข้อมูลและถ่ายทอดอัตลักษณ์ของพื้นที่ผ่านเครื่องมือที่คุ้นเคย (LINE OA) เพื่อให้ AI นำไปประมวลผลเป็นข้อมูลที่ถูกต้องและเป็นปัจจุบัน

**3. สถานะการพัฒนา:** ☑ เป็นสิ่งประดิษฐ์ใหม่ที่คิดค้นขึ้นเอง
ทีมออกแบบและพัฒนาระบบขึ้นใหม่ทั้งหมด ไม่ได้ดัดแปลงหรือต่อยอดจากซอฟต์แวร์ของผู้อื่น (อ้างถึงแพลตฟอร์มท่องเที่ยวชุมชน CBT Thailand เพียงเพื่อเปรียบเทียบให้เห็นความต่างเชิงแนวคิดเท่านั้น) จุดที่เป็นนวัตกรรมใหม่ ได้แก่ (1) AI เข้าใจผู้ใช้รายบุคคล (2) วางแผนแบบลากวาง (3) วาด Road Map บนแผนที่จริง (4) เช็คอินรับไอเทม 3 มิติ (5) หมวดท่องเที่ยวสายทำบุญ Watt's Up! **(ทั้งหมดทำงานจริงแล้ว)** (6) ระบบข้อมูลเปิด-ปิดที่ออกแบบให้ผสานข้อมูลอัตโนมัติกับการแจ้งจากคนในท้องถิ่นผ่าน LINE OA — ปัจจุบันคำนวณสถานะอัตโนมัติจากเวลาเปิด-ปิดได้แล้ว และพัฒนาฐานระบบรับแจ้งจากชุมชนผ่าน LINE OA ไว้แล้ว เตรียมนำร่องเฟสถัดไป

**4. คุณสมบัติ/คุณลักษณะเฉพาะและขอบเขตการใช้งาน**
- เว็บแอปพลิเคชันที่ใช้งานบนมือถือและคอมพิวเตอร์ผ่านเบราว์เซอร์ ไม่ต้องติดตั้งแอป **(ทำงานจริงแล้ว)**
- ปัจจุบันมีฐานข้อมูลจริง **12 สถานที่ ครอบคลุมทั้ง 6 อำเภอ** ของจังหวัดสิงห์บุรี, **6 เทศกาล/งานประจำจังหวัด**, **ตลาดท้องถิ่น 4 แห่ง** และข้อมูลวัด **6 แห่ง** ในหมวด Watt's Up! **(ทำงานจริงแล้ว)**
- **บันทึกทริปที่สร้างไว้ดูซ้ำได้** — ปัจจุบันบันทึกในเครื่องผู้ใช้ (localStorage) และพัฒนาฐานระบบ **LINE Login** เพื่อผูกบัญชีไว้แล้ว เตรียมเปิดใช้เฟสถัดไป
- **ช่องทางชุมชน (LINE Official Account):** ออกแบบให้เจ้าของร้าน/ตลาด/คนในท้องถิ่นแจ้งสถานะเปิด-ปิดและข้อมูลพื้นที่เข้าระบบ และผู้ใช้รับแจ้งเตือนเมื่อสถานะสถานที่ในแผนเปลี่ยน — **พัฒนาฐานระบบ (LINE webhook) ไว้แล้ว · นำร่องเฟสถัดไป**
- รองรับภาษาไทย (และต่อยอดภาษาอังกฤษได้)

**5. จุดเด่น/กลไกที่แตกต่าง**
- **AI Personalization:** เข้าใจว่าผู้ใช้เป็นใคร สไตล์ชีวิตแบบไหน แล้วเสนอแผนหลายรูปแบบ
- **Map-based Roadmap:** วาดเส้นทางจริงบนแผนที่ จากบ้าน → จังหวัด → แตกเป็นรายอำเภอ → ไล่จุดต่อเนื่องได้เกิน 1 วัน พร้อมแสดงโรงแรมใกล้จุดเที่ยว
- **ปรับแก้ลากวาง:** เพิ่ม/ลบ/สลับลำดับสถานที่ได้ทันที
- **ข้อมูลเปิด-ปิด (Real-time Info Sync):** (ก) คำนวณสถานะเปิด-ปิดอัตโนมัติจากเวลาทำการ/ข้อมูล Google Places ระบบแสดงป้าย "เปิดอยู่/ใกล้ปิด/ปิดแล้ว" และ AI หลีกเลี่ยงการจัดสถานที่ที่ปิด กัน "ไปถึงแล้วปิด" **(ทำงานจริงแล้ว)** (ข) ออกแบบให้ **รับแจ้งสถานะจากคนหน้างานจริงผ่าน LINE OA** (เจ้าของร้าน/ตลาด/ชุมชน เช่น ปิดกะทันหัน ติดงานวัด น้ำท่วม) มา override ข้อมูลอัตโนมัติ **(พัฒนาฐานระบบ LINE webhook ไว้แล้ว · นำร่องเฟสถัดไป)**
- **เกมิฟิเคชัน:** เช็คอินรับไอเทม 3 มิติ (ของที่ระลึก / คูปองส่วนลดแบบใช้แล้วหมดไป) **(ทำงานจริงแล้ว)**
- **Watt's Up!:** หมวดสายทำบุญ พร้อมประวัติวัดและพระเกจิดัง **(ทำงานจริงแล้ว)**
- **ระบบคำนวณค่าใช้จ่ายเป็นช่วง ฿ (ต่อคน):** ประมาณค่าใช้จ่ายการเดินทางโดยอัตโนมัติ (ค่าเดินทาง + ค่าเข้าชม + ค่าอาหาร) แสดงเป็นช่วงงบต่อคน ช่วยวางงบล่วงหน้า **(ทำงานจริงแล้ว)**
- **บันทึกทริป:** บันทึกและเรียกดูแผนที่ AI สร้างไว้ได้ทุกเมื่อ — ปัจจุบันบันทึกในเครื่อง **(ทำงานจริงแล้ว)** และพัฒนาฐานระบบ **LINE Login** เพื่อผูกบัญชีไว้แล้ว **(พัฒนาฐานระบบไว้แล้ว · นำร่องเฟสถัดไป)**
- **แจ้งเตือนผ่าน LINE:** ออกแบบให้ส่งแจ้งเตือนเมื่อสถานะสถานที่ในแผนเปลี่ยน ผ่าน LINE Messaging API **(พัฒนาฐานระบบไว้แล้ว · นำร่องเฟสถัดไป)**

**6. หลักการ วิธีการ และขั้นตอนการทำงาน**
1. ผู้ใช้ตอบคำถามสั้นๆ เพื่อสร้างโปรไฟล์/ไลฟ์สไตล์ (และเข้าสู่ระบบด้วย LINE Login เพื่อผูกบัญชี — พัฒนาฐานระบบไว้แล้ว · เปิดใช้เฟสถัดไป)
2. ส่งโปรไฟล์ผู้ใช้ + ฐานข้อมูลสถานที่ ให้โมเดล AI ประมวลผล
3. ระบบคำนวณสถานะเปิด-ปิดอัตโนมัติจากเวลาทำการ/ข้อมูล Google Places เพื่อกรองเฉพาะสถานที่ที่เปิดในช่วงเวลานั้น (และออกแบบให้ผสานสถานะที่ชุมชนแจ้งผ่าน LINE OA มา override — นำร่องเฟสถัดไป)
4. AI คืนผลเป็นชุดเส้นทาง (roadmap) หลายแบบในรูปแบบข้อมูลที่มีโครงสร้าง พร้อมช่วงค่าใช้จ่ายโดยประมาณต่อคน
5. ระบบวาดเส้นทางบนแผนที่ พร้อมแนะนำโรงแรม/ร้านอาหารใกล้เคียง และแสดงป้ายเวลาเปิด-ปิด
6. ผู้ใช้ปรับแก้แผนด้วยการลากวาง และบันทึกทริปไว้ดูซ้ำได้ (และรับแจ้งเตือนผ่าน LINE เมื่อสถานะเปลี่ยน — เฟสถัดไป)
7. เมื่อเดินทางไปถึงและเช็คอิน ระบบมอบไอเทม 3 มิติเก็บใน collection

**7. วัสดุและอุปกรณ์ที่ใช้** (เทคโนโลยี)
เว็บเฟรมเวิร์ก Next.js/React, แผนที่ MapLibre GL JS, **วาดเส้นทางบนแผนที่จริงด้วย GeoJSON** (ออกแบบให้ดึงเส้นทางถนนจริงผ่าน Google Routes API · ปัจจุบันใช้เส้นทางสาธิต/เส้นตรง), โมเดล 3 มิติ model-viewer, โมเดลภาษา Gemini API (Google Gemini 2.5 Flash), ข้อมูลเวลาเปิด-ปิดจริง Google Places API, การยืนยันตัวตน/แจ้งเตือน **LINE Login + LINE Messaging API (LINE OA)** *(พัฒนาฐานระบบไว้แล้ว · นำร่องเฟสถัดไป)*, ฐานข้อมูล/จัดเก็บ Supabase, โฮสต์บน Vercel — เครื่องมือส่วนใหญ่เป็นโอเพนซอร์ส/มีแพ็กฟรี ต้นทุนต่ำ

**8. ประโยชน์และคุณค่า**
ลดเวลาและความยุ่งยากในการวางแผนทริป เพิ่มความแม่นยำในการเดินทางด้วยข้อมูลเปิด-ปิด กระตุ้นการท่องเที่ยวเมืองรอง กระจายรายได้สู่ชุมชน/วัด/ร้านค้า ส่งเสริมการเรียนรู้ประวัติศาสตร์และวัฒนธรรมท้องถิ่น และที่สำคัญคือออกแบบให้ชุมชนเป็น "ผู้ร่วมสร้างองค์ความรู้" โดยมีส่วนร่วมดูแลข้อมูลและถ่ายทอดอัตลักษณ์ของตนเองผ่าน LINE OA ที่คุ้นเคย **(พัฒนาฐานระบบไว้แล้ว เตรียมนำร่องกับชุมชนจริง)** ทำให้ Sing Journey ใช้ AI ลดช่องว่างระหว่างเทคโนโลยีกับชุมชน ไม่ใช่แทนที่ชุมชน
