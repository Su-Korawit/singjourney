# Implement Plan — แนะนำร้านเล็ก/ร้านลับ เพื่อกระจายรายได้ถึงรากหญ้า (Hidden Gems)

> วันที่: 2026-06-29
> ที่มา: ครูเสนอเพิ่มในรอบ grill-with-docs — "ถ้าต้องการสนับสนุนการกระจายรายได้ อาจแนะนำร้านเล็ก ร้านลับ สำหรับสายกิน หรือร้านเล็ก ๆ สายอื่น เพื่อสนับสนุนไปถึงรากหญ้า"
> เป้าหมาย: ให้ AI/ระบบดัน "ร้านรายย่อยที่คนไม่ค่อยรู้จัก" เข้าไปในแผน ไม่ให้กระจุกอยู่แต่จุดดังที่คนเยอะอยู่แล้ว — ตรงกับ narrative กระจายรายได้สู่เศรษฐกิจฐานราก

## แนวคิดหลัก
"Hidden gem" = สถานที่/ร้านรายย่อยที่ (ก) เป็นกิจการชุมชน/รายเล็ก (ข) คนค้นพบน้อย (ค) อยู่ใกล้เส้นทางที่ AI วางไว้อยู่แล้ว → แทรกเป็น "ตัวเลือกเสริม" โดยไม่ทำให้ทริปยืดเกินไป

## สถานะปัจจุบัน (ของจริงในโค้ด)
- `Place.category` มี: `historical | cultural | temple | adventure | food | kids` — **ยังไม่มีแฟล็กแยก "ร้านเล็ก/ร้านลับ"**
- `Place` มี `avg_price`, `lat/lng`, `district` ครบ ใช้ทำ scoring/ระยะทางได้
- planner ฝั่ง deterministic + AI มีอยู่แล้ว (`src/lib/ai/planner.ts`) — แทรก field ใหม่เข้า prompt/post-processing ได้

## ทางเลือกการ implement (เลือกตามเวลา)

### ตัวเลือก A — Data flag + UI tag (เบาสุด ทำก่อนได้)
1. เพิ่ม field ใน `Place`: `is_local_gem: boolean` (+ optional `gem_note: string` เช่น "ร้านก๋วยเตี๋ยวเจ๊..เปิด 40 ปี") — migration + seed
2. ติดธงให้ร้านรายย่อยที่คัดมาจริง (เริ่มจากหมวด `food` ก่อน แล้วขยายสายอื่น)
3. UI: badge "ร้านลับชุมชน" บนการ์ดสถานที่ + ฟิลเตอร์ "อุดหนุนร้านเล็ก"
4. เทสต์: ฟิลเตอร์คืนเฉพาะ `is_local_gem`; การ์ดโชว์ badge

### ตัวเลือก B — Detour recommender (ได้คุณค่าจริงตาม narrative)
5. ฟังก์ชัน `suggestGemsAlongRoute(roadmap, gems, maxDetourKm)` — หา gem ที่อยู่ห่างจากเส้นทาง ≤ maxDetourKm แล้วเสนอเป็น "แวะเพิ่ม" (haversine reuse จาก cost estimator)
6. แสดงเป็น "การ์ดแนะนำ" ใต้ roadmap: "อยากอุดหนุนร้านเล็กระหว่างทางไหม?" ผู้ใช้กดเพิ่มเข้าทริปได้ (ใช้ add/remove helper ที่มีอยู่)
7. เทสต์: gem นอกรัศมีไม่ถูกเสนอ; gem ในรัศมีถูกเสนอเรียงตาม detour น้อย→มาก

### ตัวเลือก C — AI-aware fairness (ขั้นสูง)
8. ใส่กติกาใน planner prompt: ในแต่ละวันให้มี gem อย่างน้อย 1 จุดถ้ามีในพื้นที่ และอย่าจัดแต่จุดดังซ้ำ ๆ
9. (อนาคต) ผูกกับ LINE OA loop: ร้านที่ลงทะเบียนผ่าน LINE OA → ติดธง gem อัตโนมัติ เชื่อมกับ [แผน LINE OA](2026-06-29-line-oa-community-loop-activation.md)

## ลำดับแนะนำ
A → B → (C ถ้าเวลาเหลือ). ตัวเลือก A ทำให้เคลม "แนะนำร้านเล็ก/ร้านลับ" ในฟอร์ม/infographic เป็นจริงได้เร็ว (ตอนนี้ใส่ใน infographic ข้อ 8 แล้วในฐานะแนวคิด)

## เกณฑ์สำเร็จ
- [ ] มีฟิลเตอร์/badge ร้านเล็กชุมชน และมี seed จริงอย่างน้อย ~5 ร้าน
- [ ] ระบบเสนอ gem ระหว่างทางได้โดยไม่ทำทริปยืดเกิน maxDetourKm
- [ ] ผู้ใช้กดเพิ่ม gem เข้าทริปได้

## ข้อควรระวัง
- เกณฑ์ "ร้านลับ/รายย่อย" ต้องคัดด้วยคนจริง อย่าให้ AI มั่ว — ไม่งั้นเสี่ยงข้อมูลผิด/ร้านปิดไปแล้ว (โยงกับ Real-time Info Sync)
- อย่าดันร้านเล็กจนทริปเสียคุณภาพ — เป็น "ตัวเลือกเสริม" ไม่ใช่บังคับ
