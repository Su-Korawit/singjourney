# Sing Journey — บันทึกการตัดสินใจ (จาก Grilling Session)

## คำศัพท์ (Glossary)
- **Roadmap / Road Map** — เส้นทางท่องเที่ยวที่ระบบสร้าง วาดบนแผนที่จริง เริ่มจากบ้าน → จังหวัด → แตกรายอำเภอ → ไล่จุดต่อเนื่องได้เกิน 1 วัน
- **Profile** — ข้อมูลตัวตน/ไลฟ์สไตล์ผู้ใช้ (เช่น แม่เลี้ยงเดี่ยว งบจำกัด ลูกเล็ก) ใช้ป้อน AI วางแผน
- **Watt's Up!** — หมวดท่องเที่ยวสายทำบุญ รวมวัด + ประวัติวัด/พระเกจิดัง (เล่นคำ Watt/วัด)
- **ไอเทม 3D** — ของรางวัลจากการเช็คอิน แบ่งเป็น ของที่ระลึก (เก็บถาวร) และ คูปองส่วนลด (consumable — ใช้แล้วหมด รับซ้ำไม่ได้)
- **Check-in** — การยืนยันว่าผู้ใช้ไปถึงสถานที่จริง เพื่อรับไอเทม
- **Real-time Info Sync** — ระบบดึงข้อมูลจริงแบบเรียลไทม์ (เวลาเปิด-ปิด + สถานะเปิด-ปิดชั่วคราว/ถาวร) ของแต่ละสถานที่ผ่าน Google Places API เพื่อความแม่นยำในการเดินทาง กัน "ไปถึงแล้วปิด"

## การตัดสินใจที่ล็อกแล้ว (ADR)
1. **เริ่มใหม่ทั้งหมด** — ไม่มีโค้ดเดิม CBT Thailand เป็นแค่ benchmark/แรงบันดาลใจ
2. **กลุ่มประกวด: กลุ่มที่ 5** การท่องเที่ยวและเศรษฐกิจสร้างสรรค์
3. **ระดับ: ม.ปลายเท่านั้น** (เงินรางวัลชนะเลิศ 50,000 / รอง 1 = 40,000 / รอง 2 = 30,000)
4. **คืนนี้ทำเอกสารส่งก่อน** + เขียน Implementation Plan เตรียม build เดโมทีหลัง
5. **Stack:** Next.js + MapLibre + model-viewer + Claude API (opus-4-8) + Supabase + LINE Login
6. **ไอเทม 3D = โมเดลหมุนได้ในเว็บ** (ไม่ทำ AR ในเดโม)
7. **ตัดการจองโรงแรม + โค้ดส่วนลดจริงออกจากเดโม** (แสดงเป็น mock UI)
8. **เพิ่มระบบ Real-time Info Sync** — ดึงเวลาเปิด-ปิดจริงผ่าน Google Places API, cache ใน Supabase (sync วันละ 1–2 ครั้งผ่าน Cron), ใช้กรองสถานที่ใน AI planner + แสดงป้ายสถานะ — key อยู่ฝั่ง server เท่านั้น

## คำถามที่ยังเปิด (ตอบก่อน build)
- ชื่อทีม + ชื่อนักเรียน/อาจารย์ (ใส่ในฟอร์ม 1/3/4) — ยังไม่ระบุ
- ชื่อภาษาไทยของผลงาน — ผมเสนอ "สิงห์เจอร์นีย์" ปรับได้
- รายชื่อสถานที่จริง + พิกัด ของแต่ละอำเภอในสิงห์บุรี
- งบประมาณที่ใช้พัฒนา (ฟอร์ม 1 ข้อ 9)
